traceFile=path 属性

機能

DerbyネットワーククライアントがJDBCトレース情報を書き出す先のファイルを指定します。絶対パス名を指定しなかった場合、ファイルはderby.system.homeディレクトリに置かれます。(詳細についてはDerby Developer's Guideの「システムディレクトリを定義する」を参照してください。)

traceFile=pathとtraceFileAppend=trueの両方を指定した場合、指定したファイルが既存であれば、出力はそのファイルに追加されます。もし、traceFile=pathを指定してtraceFileAppend=trueを指定しなかった場合、ファイルは上書きされ、前に書かれていた内容は失われます。

トレースに関するより詳細な情報は、Derby Server and Administration Guideの「ネットワーククライアントのトレース」を参照してください。 トレースに関する他の属性は、 traceDirectory=pathと traceLevel=valueを参照してください。

他の属性との組み合わせ

この属性はほかの属性と組み合わせることができます。

例

-- 新規データベースでトレースを有効にする。
jdbc:derby://localhost:1527/mydb;create=true;traceFile=trace.out
親トピック: データベース接続URLに属性を指定する。
